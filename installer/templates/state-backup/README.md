# agent-state-backup

Live state backup for [tsevmenko/telegram-claude-platform](https://github.com/tsevmenko/telegram-claude-platform) agents.

This repository is **private** because it contains operator-specific data:
USER profiles, brand voice rules, content strategy, scraped competitor
archives, custom skills built by agents.

## What is here

- `<agent>/` (one per agent: `leto/`, `tyrion/`, `varys/`, `vesna/`, plus
  any future agents) — workspace mirrors with CLAUDE.md, core/ memory
  layers, custom skills, scheduled cron entries.
- `shared/ov-snapshot-YYYY-MM-DD.sql.gz` — daily L4 OpenViking semantic-
  memory dumps (cross-agent shared brain), last 14 retained.

## What is NOT here

- Source code (lives at `tsevmenko/telegram-claude-platform`, public, MIT).
- Secrets (API keys, OAuth tokens, webhook tokens) — strict `.gitignore`
  + pre-commit secret-scanner.
- Logs (raw), node_modules, .venv, claude CLI session JSONLs.

## How it stays in sync

`tools/state-backup-runner.sh` runs every 4 hours via cron, performs
rsync from VPS workspaces → this repo, runs the secret-scanner against
staged files, and pushes.

`tools/openviking-snapshot.sh` runs nightly to dump the L4 SQLite db
into `shared/`.

## Restore on a fresh VPS

After running the platform's `install.sh` to build a fresh deployment,
restore agent state with:

```bash
git clone --depth 1 git@github-state-backup:<owner>/agent-state-backup.git \
    /tmp/restore
sudo bash /tmp/restore/tools/restore-from-backup.sh   # walks the operator
```

(`restore-from-backup.sh` is generated by the installer step
`80-state-backup.sh` of the main platform repo.)

## Schedule

| When (UTC) | What |
|---|---|
| every 4h (`0 */4 * * *`) | state-backup-runner — rsync + commit + push |
| daily 03:00 | openviking-snapshot — SQLite dump |

## Security model

- Operator's GitHub account → repo is private, only operator + invited
  collaborators can read.
- VPS pushes via an SSH **deploy key** (write-scope, scoped to this one
  repo only). Compromise of VPS gives an attacker access ONLY to push
  history of this repo, not to operator's other GitHub assets.
- Pre-commit secret-scanner refuses any commit that introduces a
  token-shaped string (regex match on `sk-ant-`, `gsk_`, `ghp_`, `xoxb-`,
  `pk_*_*` PATs, `Bearer <long>`, `JWT eyJ*`, AWS keys, etc.).
- Allowlist for known false-positives lives in
  `tools/secret-scanner.sh`; add carefully.
